📖 插件开发规范

📋 概述

本规范定义了 RS Log Analyzer 的插件格式，AI 或开发者可以按照此格式自动生成插件文件。

📁 文件位置

插件文件放置在 plugins/ 文件夹中，格式为 .js 文件。

🔧 插件格式

1. 文件命名规则

<插件标识>.js

例如：
- sport_analyzer.js
- ota_v2.js
- ble_connect.js
- neck_health.js

2. 全局变量命名

window.__PLUGIN_<插件标识> = {
  // 插件内容
};

命名规范：
- 插件标识使用小写字母和下划线
- 全局变量前缀固定为 __PLUGIN_
- 例如：window.__PLUGIN_sport_analyzer = { ... };

📦 插件 JSON 结构

window.__PLUGIN_<标识> = {
  "meta": {
    "name": "插件显示名称",
    "version": "1.0.0",
    "author": "作者名称",
    "description": "插件功能描述"
  },
  "modules": [
    {
      "id": "模块唯一标识",
      "name": "模块显示名称",
      "icon": "图标或Emoji",
      "category": "分类标识",
      "description": "模块功能描述（可选）",
      "patterns": [
        {
          "id": "模式唯一标识",
          "match": "正则表达式字符串",
          "description": "模式描述",
          "variables": [
            {
              "name": "变量名",
              "extract": "\\1",
              "type": "number|string",
              "unit": "单位（可选）"
            }
          ]
        }
      ]
    }
  ],
  "flows": [
    {
      "id": "步骤唯一标识",
      "name": "步骤显示名称",
      "pattern": "对应的 pattern id",
      "description": "步骤描述"
    }
  ]
};

📖 字段详细说明

meta（元数据）

字段类型必填说明
namestring✅插件显示名称
versionstring✅版本号，如 "1.0.0"
authorstring✅作者名称
descriptionstring✅插件功能描述

modules（模块数组）

每个模块包含一组相关的日志匹配模式。

字段类型必填说明
idstring✅模块唯一标识（小写字母+下划线）
namestring✅模块显示名称
iconstring✅图标（建议使用Emoji，如 📦）
categorystring✅分类标识
descriptionstring⬜模块功能描述
patternsarray✅日志匹配模式数组

常用分类：bluetooth, sensor, algorithm, flash, system, sport, ota, error, info, tws

patterns（模式数组）

定义如何匹配日志行并提取变量。

字段类型必填说明
idstring✅模式唯一标识（小写字母+下划线）
matchstring✅正则表达式（注意转义：\\d+ \\w+ 等）
descriptionstring⬜模式描述
variablesarray⬜变量提取配置数组

variables（变量数组）

从匹配结果中提取数据。

字段类型必填说明
namestring✅变量名（英文，下划线分隔）
extractstring✅提取表达式（如 "\\1", "\\2"）
typestring✅数据类型（number 或 string）
unitstring⬜单位（如 ms, bpm, °C, bytes）

flows（流程数组）

定义日志分析的标准流程步骤。

字段类型必填说明
idstring✅步骤唯一标识
namestring✅步骤显示名称
patternstring✅对应的 pattern id
descriptionstring⬜步骤描述

💡 正则表达式规范

常用正则模式

// 匹配数字
"(\\d+)"                    // 匹配任意数字
"(\\d+) ms"                 // 匹配数字+单位
"(\\d+\\.\\d+)"             // 匹配小数

// 匹配十六进制
"(0x[0-9A-Fa-f]+)"          // 匹配 0x1234AB
"(0x[0-9A-F]{4})"           // 匹配 0xABCD（固定4位）

// 匹配字符串
"([a-zA-Z_]+)"              // 匹配字母和下划线
"([\\w]+)"                  // 匹配单词字符
"(.+)"                      // 匹配任意字符（贪婪）

// 匹配可选内容
"(success|fail|\\d+)"       // 匹配 success 或 fail 或数字
"(true|false|0|1)"          // 匹配布尔值

// 匹配MAC地址
"([0-9A-Fa-f:]{17})"        // 匹配 AA:BB:CC:DD

📝 完整示例

window.__PLUGIN_bluetooth = {
  "meta": {
    "name": "蓝牙连接分析器",
    "version": "1.0.0",
    "author": "Your Name",
    "description": "分析蓝牙连接和断开的日志"
  },
  "modules": [
    {
      "id": "ble_connect",
      "name": "BLE连接",
      "icon": "🔗",
      "category": "bluetooth",
      "description": "蓝牙低功耗连接过程",
      "patterns": [
        {
          "id": "ble_connected",
          "match": "BLE connected, addr=([0-9A-Fa-f:]{17}), conn_id=(\\d+)",
          "description": "BLE设备连接成功",
          "variables": [
            { "name": "mac_address", "extract": "\\1", "type": "string" },
            { "name": "connection_id", "extract": "\\2", "type": "number" }
          ]
        },
        {
          "id": "ble_disconnected",
          "match": "BLE disconnected, reason=(\\d+)",
          "description": "BLE设备断开连接",
          "variables": [
            { "name": "disconnect_reason", "extract": "\\1", "type": "number" }
          ]
        }
      ]
    }
  ],
  "flows": [
    { "id": "step_1", "name": "设备连接", "pattern": "ble_connected" },
    { "id": "step_2", "name": "设备断开", "pattern": "ble_disconnected" }
  ]
};

🚀 使用步骤

  1. 将上述代码保存为 plugins/bluetooth.js
  2. 在 index.html 中添加 <script src="plugins/bluetooth.js">
  3. 在 loadPluginsFromFolder() 函数中添加插件映射
  4. 刷新页面即可使用新插件
